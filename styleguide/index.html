UI Library

Introduction

This is an internal styleguide for developing Maps Sdk for Web V6 examples. All the assets mentioned in this styleguide can be found under V6/assets/ui-library on Github.

Usage

Using stylesheet link

The easiest way to use this library is to include the index.css file in the <head> section of your HTML page. Please remember that the ui-library assets should be hosted with your own assets. 


                This index.css stylesheet contains all the components and classes bundled together for your convenience.

                Please note that you can also import the assets individually if you don't need all the components mentioned in this styleguide.

                

                    In this styleguide we just showcase the components available and their styles, we don't show how to implement the interactions.
                    Please refer to our examples to see the implementations, we also provide there some utility classes to work with these components.
                


            

            

                
Layout


                
There are two general approaches for creating controls in examples:

                
    
                    
  • Side panel - for complex examples with lots of options and/or results list (eg. Search, Routing options)
    • 
                    
  • Overlay panel - for simple examples with a few options (eg. Styles and Layers, Traffic Flow)
    • 
                


                
Side panel


                When using a side panel avoid using other panels over the map, all controls should be contained in a single panel.
                The Side panel can be folded using the fold button. When it's folded, the tt-reveal-sidepanel should be added to map container.

                

                    

                    

                    


                


                

                

                
Most important params inputs (eg. those which are required for request and not filled) should go on the top of options list. Options container needs to be unfloded (initialy).


                
Overlay panel


                Overlay panel contains minimal set of param controls (up to 4). If you have more than 4, search input, route input or result list, use side panel instead. Panel should be always placed in top-left corner of the map.

FAQ

            
When should I use foldable?

            
Foldable should be used for examples which contain more than one control. Valid example for not using foldable is Resize Map.

            
When should I use side panel header (.tt-side-panel__header)?

            
Side panel header is used to display most important parameters. For example in routing examples, place there inputs for starting/finish points (wrapped in .tt-input-icon)
